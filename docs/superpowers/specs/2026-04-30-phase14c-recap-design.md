# Phase 14c — Recap: `/recap` command + auto away-summary card + autoDream consolidation

**Date:** 2026-04-30
**Status:** Spec
**Depends on:** `2026-04-30-phase14-foundation-design.md` (EventBus, ProgressTracker, forkedAgent, on-disk layout `~/.nuka/recaps/`, `~/.nuka/events/<sess>.ndjson`), `2026-04-30-phase14a-swarm-design.md` (swarm activity is one of the 9 recap fields)
**Author:** Brainstorming session 2026-04-30

## 1. Problem

Nuka sessions today are write-only — once tasks complete and the user closes the REPL (or even just steps away for 30 minutes), there is no surface that says "this is what we did". Specifically:

- No **`/recap` slash command** at all (the slash registry doesn't have one)
- No **auto away-summary card** when the user comes back after idle (Claude Code's pattern: 1–3 sentence "while you were away" hint, generated by a small fast model from the last N messages)
- No **persisted recap** — long projects can't be resumed across sessions with a "last meeting notes" view
- No **autoDream** memory-consolidation analog — the `~/.nuka/memdir/` (memory directory) collects entries forever; nothing rolls them up periodically

Phase14c adds these surfaces. All four use the foundation's `forkedAgent` for prompt-cache reuse (a recap is fork-on-the-current-session, just like Claude Code's `awaySummary`).

User-confirmed scope (brainstorm 2026-04-30): **(a) command-form `/recap` + (b) auto away-card** for the surface, **all 9 fields**, plus persistence under `~/.nuka/recaps/`.

## 2. Goals

1. **`/recap` slash command** — produces a structured recap of the current session (or a sub-window), prints in the conversation, and writes to `~/.nuka/recaps/<YYYY-MM-DD>-<sessionId>.md`. Default scope = full current session; flags: `--since 1h` / `--since 30m` / `--agent <name>` / `--pipeline <id>`.
2. **9 recap fields, all rendered**:
   1. Completed tasks (from EventBus replay of `task.state → completed`)
   2. In-progress / failed tasks
   3. File diffs by agent (joined with checkpoint log if available)
   4. Tool-call timeline (compressed: consecutive Read/Grep collapsed)
   5. Swarm message digest (top-10 envelopes by importance heuristic)
   6. Pipeline node states (which stages ran, which failed)
   7. Token / cost rollup per model + per agent
   8. Next-step suggestion (small fast model, 1 paragraph)
   9. Key decisions (extracted from messages tagged with brainstorm/plan/handoff)
3. **Auto away-summary card** — when the user has been idle ≥ 5 minutes (no input event), on next keystroke, render a 1–3 sentence card at the top of the next conversation update. Generated by `runForkedAgent` against `getSmallFastModel()` (or default model if no fast model configured). Card is dismissable with `Esc`.
4. **Persisted recaps** — every successful `/recap` writes to `~/.nuka/recaps/`. File is markdown; first line is YAML front-matter `{ session, generatedAt, scope }`. Retention: 90 days (foundation §5.7).
5. **autoDream** — periodic background memory consolidation. When (`hours_since_last_consolidation ≥ 6h` AND `new_session_count ≥ 3`), spawn a `dream` Task that forks the agent to read recent memdir entries and produce a single consolidated entry. Time-gate first to be cheap.
6. **Idle detection** — robust to `Esc` interrupts, mouse focus loss, and SSH connection lapses. Uses last-input timestamp from the input event source; if the user's terminal hasn't been the foreground tab (TIOCNOTTY proxy or simply no input event), still considered "idle".

## 3. Non-Goals

- ❌ No real-time recap (the live data is what `/monitor` shows; recap is a *digest*)
- ❌ No cross-session aggregation in `/recap` itself (only current session); cross-session is the job of autoDream
- ❌ No automatic on-exit recap (user must explicitly run `/recap` to save; rationale: most exits are quick "let me try a thing", not project completion)
- ❌ No team / shared recaps — recap is per-session, per-user
- ❌ No backfill of recaps for already-finished sessions without event log (the event flusher must have been on)
- ❌ Auto away-card is **opt-out**, not gated behind a feature flag; user can disable via `~/.nuka/config.yaml` `recap.awayCard: false`

```

                │  └───────────────────────────────────────────────┘  │
                │                                                    │

   Sources of truth:
     - EventBus ring (live + last 1024)
     - events/<sess>.ndjson (durable, opt-in)
     - ~/.nuka/memdir/ (existing memory directory)
     - Session.messages (last N for fast-model away summary)
```

## 5. Data schemas

### 5.1 RecapDoc

```ts
type RecapScope =
  | { kind: 'full' }
  | { kind: 'since'; ms: number }
  | { kind: 'agent'; name: string }
  | { kind: 'pipeline'; id: string }

type RecapDoc = {
  session: string
  generatedAt: number
  scope: RecapScope
  fields: {
    completed: { id: string; description: string; durationMs: number; agentName?: string }[]
    inFlight:  { id: string; state: TaskState; description: string }[]
    fileDiffs: { agentName: string; path: string; added: number; removed: number }[]
    toolTimeline: { t: number; toolName: string; collapsedCount: number; sessionId: string }[]
    messages: { id: string; from: string; to: string; summary: string; t: number }[]   // top 10
    pipelines: { pipelineId: string; nodes: { id: string; status: TaskState; agent: string }[] }[]
    tokens: { perAgent: Record<string, { in: number; out: number }>; cost?: number }
    nextStep: string
    keyDecisions: { source: 'brainstorm' | 'plan' | 'handoff'; text: string; t: number }[]
  }
}
```

## 6. Component contracts

### 6.1 `/recap` slash command — `src/slash/recap.ts`

```ts
export const recapCommand: SlashCommand = {
  name: 'recap',
  description: 'Generate a recap of the current session',
  async handler(ctx, args) {
    const scope = parseScope(args)              // 'full' | 'since 1h' | etc.
    const doc = await buildRecap({
      sessionId: ctx.session.id,
      scope,
      bus: ctx.bus,
      events: ctx.eventLog,                     // optional reader of events/<sess>.ndjson
      tasks: ctx.taskManager,
      provider: ctx.providerResolver,
      session: ctx.session,
    })
    const md = renderMarkdown(doc)
    ctx.printAssistant(md)
    await persistRecap(ctx.home, doc)
    return { ok: true }
  },
}
```

### 6.2 Recap builder — `src/core/recap/builder.ts` (new)

```ts
export async function buildRecap(opts: BuildRecapOpts): Promise<RecapDoc> {
  const events = collectEvents(opts)            // bus replay + ndjson if available
  const fields: RecapDoc['fields'] = {
    completed:    fields.completed(events),
    inFlight:     fields.inFlight(events),
    fileDiffs:    fields.fileDiffs(events, opts.checkpointLog),
    toolTimeline: fields.toolTimeline(events),
    messages:     fields.messages(events),
    pipelines:    fields.pipelines(events),
    tokens:       fields.tokens(events),
    keyDecisions: fields.keyDecisions(events),
    nextStep:     await fields.nextStep(events, opts),    // forked agent
  }
  return { session: opts.sessionId, generatedAt: Date.now(), scope: opts.scope, fields }
}
```

Each `fields.*` is a pure reducer over event records. `fields.nextStep` is the single async one — calls `runForkedAgent` with a 200-token budget.

### 6.4 Recap persistence — `src/core/recap/persist.ts` (new)

```ts
export async function persistRecap(home: string, doc: RecapDoc): Promise<string> {
  const dir = recapsDir(home)
  await fsp.mkdir(dir, { recursive: true })
  const date = new Date(doc.generatedAt).toISOString().slice(0, 10)
  const file = path.join(dir, `${date}-${doc.session}.md`)
  await fsp.writeFile(file, renderMarkdown(doc), 'utf8')
  return file
}
```

### 6.5 autoDream — `src/core/recap/autoDream.ts` (new)

```ts
export type AutoDreamOpts = {
  home: string
  bus: EventBus
  tasks: TaskManager
  session: Session
  provider: ProviderResolver
  config: { minHours: number; minSessions: number }
}

export function initAutoDream(opts: AutoDreamOpts): { tick: () => Promise<void>; stop: () => void } {
  // Returns a tick() callable — the bootstrap installs it on a 30-min interval.
}
```

Tick logic (cheapest gate first, then expensive):
1. Read `~/.nuka/memdir/.dream.meta.json` for `lastConsolidatedAt`. If `now - that < minHours*3600_000`, return.
2. Count session transcripts touched since `lastConsolidatedAt`. If < `minSessions`, return.
3. Try-acquire `.dream.lock` (atomic create-exclusive). If exists, return.
4. Enqueue a `Task<dream>` with `consolidationPrompt` from `core/recap/consolidationPrompt.ts`.
5. Dream runner uses `runForkedAgent` reading recent memdir entries; writes a consolidated entry; deletes superseded entries.
6. On completion, update `lastConsolidatedAt` and release lock.

`run-dream.ts` (was a stub from foundation M1) gets its full body here.

## 7. Testing strategy

| Area | Test type | Coverage |
|------|-----------|----------|
| Each `fields.*` reducer | unit | empty input → empty output; mixed event types → correct rollup; ordering by t |
| `fields.nextStep` | unit + msw | small-fast-model called; returns single paragraph; truncates > 500 chars |

| `renderMarkdown` | snapshot | golden-file render of a fixture RecapDoc |
| `persistRecap` | unit + tmpdir | file written under `recaps/<date>-<sess>.md`; idempotent on rerun |

| `--since 1h` parsing | unit | `1h`, `30m`, `90s` parsed; bad values rejected |
| Idle watcher | unit + fake clock | fires `onAway` at threshold; `onReturn` only after `onAway`; threshold reset on input |
| AwaySummaryCard | ink-testing-library | renders in 1-3 sentences; Esc dismisses; auto-fade after 30s |
| autoDream gating | unit + fake clock | does NOT fire below thresholds; DOES fire when both met; lock prevents concurrent |

## 9. Risks

| Risk | Mitigation |
|------|------------|
| `/recap` is slow when events ring buffer is large + small-fast-model call adds 2s | Reducers run on cached snapshot; `forkedAgent` next-step is 200-token cap; user sees a "Generating recap…" line while it runs |
| Away-summary card distracting at 5-min idle if user just left for coffee | User can set `awayThresholdMinutes` higher; auto-fades after 30s |
| Markdown render explodes if event log has 10k entries | Top-10 messages rule; tool timeline collapses consecutive Read/Grep; cap each section at 50 entries |
| autoDream eats memory when memdir is huge | Reads in 100-entry chunks; merges incrementally; budget 4096 output tokens |
| Idle detector races with input-event arrival on very slow ssh | Watcher uses monotonic clock; threshold check is on every input — no missed reset |
| Dream lock orphaned on crash | Lock has a `startedAt` field; > 5 minutes old → ignored and overwritten |
| Cost calculation diverges from `/cost` | Reuses the existing `cost-tracker` rates table; no separate model rate map |
| nextStep field hallucinates a feature that doesn't exist | Prompt is constrained to "concrete next step from the actual transcript only"; verified manually in M2 snapshot fixtures |
