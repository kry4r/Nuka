## M1 — Onboarding (tasks 7.1, 7.2-glue)

### 7.1.a Provider templates + probes
**Files:** `src/core/onboarding/templates.ts`, `src/core/onboarding/providerProbe.ts`. Tests: `test/core/onboarding/probe.test.ts`.

Contract:
```ts
export type ProviderTemplate = {
  id: 'anthropic'|'openai'
  type: string
  defaultModel: string
  defaultModels: string[]
  apiKeyEnvVar: string
  helpUrl: string
}
export const PROVIDER_TEMPLATES: ProviderTemplate[]
export async function probeProvider(t: ProviderTemplate, key: string): Promise<{ok:true; models?: string[]} | {ok:false; reason:string}>
```

Acceptance:
1. Two templates exported (anthropic, openai).
2. `probeProvider` for openai with mocked `/v1/models` 401 → `{ok:false}`; with 200+payload → `{ok:true, models:[…]}`.
3. `probeProvider` for anthropic with mocked 200 → `{ok:true}`; with 401 → `{ok:false}`.

### 7.1.b Wizard state machine
**Files:** `src/core/onboarding/wizard.ts`. Tests: `test/core/onboarding/wizard.test.ts`.

Pure reducer:
```ts
export function reducer(state: WizardState, ev: WizardEvent): WizardState
```
Events: `start`, `pickedProvider(t)`, `enteredKey(s)`, `pickedModel(m)`, `probeOk(models?)`, `probeErr(s)`, `back`, `cancel`.

Acceptance:
1. `welcome` + `start` → `pickProvider`.
2. `pickProvider` + `pickedProvider(t)` → `apiKey`.
3. `apiKey` + `enteredKey('sk-…')` → `pickModel` (using template's defaultModels until probe overrides).
4. `pickModel` + `pickedModel('m')` → `verifying`.
5. `verifying` + `probeErr('bad key')` → `error{retryFrom:'apiKey'}`.
6. `error` + `back` → previous state per `retryFrom`.

### 7.1.c TUI screens
**Files:** `src/tui/Onboarding/Wizard.tsx`, `Steps/PickProvider.tsx`, `Steps/EnterKey.tsx`, `Steps/PickModel.tsx`, `Steps/Verifying.tsx`, `Steps/Done.tsx`. Tests: `test/tui/Onboarding/wizard.test.tsx` (ink-testing-library).

Each step is a controlled component over the wizard state. Esc → cancel; Enter advances; ↑↓ navigates lists.

Acceptance:
1. Render `Wizard` with state `welcome` → snapshot shows banner + "press Enter".
2. Press Enter → state transitions to `pickProvider`; rendered list shows 2 entries.
3. Pick anthropic, type `sk-ant-test`, Enter → state goes to `pickModel`.
4. Cancel via Esc → onCancel callback fired with no config.

### 7.1.d Wire into cli.tsx + `nuka init` subcommand
**Files:** `src/cli.tsx`, `src/slash/config.ts`, `src/slash/model.ts`.

- `argv[0] === 'init'` → render `Wizard`, await result, write to `~/.nuka/config.yaml`, exit 0.
- In offline mode TUI: when user types `/config` and providers.length === 0, mount `Wizard` over the prompt; on save, hot-swap into `ProviderResolver` and start a new session with the new provider as active.

Acceptance:
1. `nuka init` from a clean home → completes wizard → `~/.nuka/config.yaml` contains the provider.
2. Offline TUI + `/config` → wizard mounts → save → next prompt routes to the new provider.
3. Existing tests still pass.

### 7.2 Offline-mode polish (already largely landed)
**Files:** `src/cli.tsx` (verify), add unit test for offline-startup path.

Acceptance:
1. Test asserts `runInteractive` does not exit when config has zero providers.

## M2 — Cost tracker + auto-memory (tasks 7.3, 7.4)

### 7.3.a Pricing table + tracker
**Files:** `src/core/cost/pricing.ts`, `src/core/cost/tracker.ts`. Tests: `test/core/cost/tracker.test.ts`.

```ts
export type Usage = { input: number; output: number; cacheCreate?: number; cacheRead?: number }
export class CostTracker {
  record(model: string, sessionId: string, usage: Usage, ts?: number): void
  current(sessionId: string): Aggregate
  today(): Aggregate
  allTime(): Aggregate
  toUsd(model: string, agg: Aggregate): number | undefined
}
```

Pricing seed: claude-opus-4-7, claude-sonnet-4-6, claude-haiku-4-5; gpt-5, gpt-4o, gpt-4o-mini.

Acceptance:
1. `record('claude-opus-4-7', s, {input:1000, output:500})` → `current(s)` reflects it.
2. `toUsd` returns ≥ 0 for known models; `undefined` for unknown.
3. Lifetime + today aggregates partition correctly across `ts`.

### 7.3.b Persistence
**Files:** `src/core/cost/persist.ts`. Tests: `test/core/cost/persist.test.ts`.

JSON file at `~/.nuka/cost.json`. Atomic write (tmp + rename). Schema `{version:1, entries: CostEntry[]}`. Cap entries at 10k; drop oldest.

### 7.3.c Wire to agent loop + `/cost` slash
**Files:** `src/core/agent/loop.ts` (after each assistant message, call `tracker.record`); `src/slash/cost.ts`. Tests: extend `test/core/agent/loop.test.ts`.

`/cost` prints the table from §5.2 of the spec.

Acceptance:
1. Mock provider returning usage on a turn → tracker has an entry.
2. `/cost` output contains "$" plus a positive number after a recorded turn with a known model.

### 7.4.a Memdir parser + storage
**Files:** `src/core/memdir/parser.ts`, `src/core/memdir/index.ts`. Tests: `test/core/memdir/parser.test.ts`.

Each entry: YAML frontmatter `{ts, sessionId, keywords[], score?}` + body.
`MemdirIndex.load(cwd)` returns `Entry[]`. `append(cwd, entry)` is atomic.

### 7.4.b Synth + relevance
**Files:** `src/core/memdir/synth.ts`, `src/core/memdir/relevance.ts`. Tests: `test/core/memdir/relevance.test.ts` (synth tested via mock provider in integration test).

`synth(turns, provider, model)` → `Entry`. Returns null on failure. 5-second timeout.

`findRelevant(entries, queryTokens, k=5)` → top-K via simple TF-IDF over `keywords` + body words.

### 7.4.c Hooks: session-end synth + system-prompt injection
**Files:** `src/core/agent/loop.ts`, `src/core/agent/systemPrompt.ts` (or wherever system prompt is assembled), `src/core/compact/auto.ts`, `src/cli.tsx` (SIGINT path). `src/slash/memdir.ts` for `list/clear/compact`.

Acceptance:
1. After `runAgentLoop` finishes a multi-turn session and SIGINT/`/exit` fires, `MEMORY.md` for the cwd grows by 1 entry (mocked provider).
2. Next `runAgentLoop` invocation includes a `## Memory` section in the system prompt with at least one of the entries (when relevance > 0).
3. `/memdir clear` empties the file; `/memdir list` enumerates entries.

## M3 — Vim + HUD + Chinese README (tasks 7.5, 7.6, 7.7)

### 7.5.a Vim controller
**Files:** `src/core/vim/{mode,motions,operators,textObjects,controller}.ts`. Tests: `test/core/vim/controller.test.ts`.

Pure functions over a buffer model `{lines: string[]; cursor: {row,col}; mode}`. Controller is the keystroke→buffer reducer.

Acceptance (table-driven):
1. `dw` from `|hello world` → `|world`.
2. `ci"` inside `"foo bar"` → enter insert with `""` and cursor inside.
3. `0` and `$` jump to line bounds.
4. `.` repeats the last op.
5. `v$d` deletes to EOL.

Cap test count ~25.
