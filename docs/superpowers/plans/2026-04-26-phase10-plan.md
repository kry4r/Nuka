## M2 — Task system

### 10.3.a Types + manager + bash runner
**Files:** `src/core/tasks/{types,manager,run-bash,persist}.ts`. Tests: `test/core/tasks/{manager,run-bash}.test.ts`.

Acceptance:
1. `enqueue({kind:'local_bash', spec:{cmd:'echo hi'}})` returns task; on completion outputFile contains `hi\n`; state `completed`.
2. `cancel(id)` SIGTERMs; eventually transitions to `killed`.
3. `list()` reflects all tasks; `on('change')` fires per transition.

### 10.3.b Agent + monitor-mcp runners
**Files:** `src/core/tasks/run-agent.ts`, `src/core/tasks/monitor-mcp.ts`. Tests: `test/core/tasks/run-agent.test.ts` (mock dispatch), `test/core/tasks/monitor-mcp.test.ts` (mock client).

Acceptance:
1. `local_agent` task drives a mocked `dispatchAgent` and persists its `assistant` chunks to outputFile.
2. `monitor_mcp` subscribes to a mock client's `progress` event stream and completes on `final`.

### 10.3.c Slash + HUD wiring
**Files:** `src/slash/tasks.ts`, `src/tui/Status/Hud.tsx` (extend). Tests: `test/slash/tasks.test.ts`, `test/tui/Status/Hud.test.tsx` (extend).

Acceptance:
1. `/tasks` lists running tasks with id/kind/state.
2. `/tasks show <id>` prints last 50 output lines.
3. `/tasks cancel <id>` succeeds.
4. HUD shows `tasks N` when N > 0.
