### M1 — StatsView async-timing flake fix  (model: sonnet)

**Worktree:** `phase13-m1` off `main`.
**Goal:** the `harness.test.tsx > /stats opens dialog` test passes deterministically.

**Pre-flight grounding**: read `src/tui/Stats/StatsView.tsx`, `test/tui/testing/harness.test.tsx` lines around `/stats opens dialog`, and `src/core/cost/tracker.ts` `byProvider()`. Identify whether `Loading…` is sync-renderable as `(no data yet)` when the result set is empty.

Deliverables:
1. Diagnose the race: log the actual timing or trace the `Loading…` → `no data yet` transition.
2. Fix at the right layer:
   - Preferred: render `(no data yet)` synchronously when `byProvider()` returns synchronously-resolvable empty, drop the `Loading…` placeholder for that path.
   - Fallback: if the byProvider call truly is async-only, update the harness test to `waitFor({ contains: 'no data yet' })` rather than `pop()`-ing immediately.
3. Add a regression test that runs the same flow with `waitFor` so the assertion is robust regardless of internal timing.
4. Run `npx vitest run test/tui/testing/harness.test.tsx --repeat 10` (or equivalent) and confirm 10 consecutive passes.

Acceptance: full vitest suite green (1147+ passed); `npm run build` clean. The `/stats opens dialog` test passes 10× in a row.

### M2 — Welcome screen redesign (Logo+hero + Updates + Recent)  (depends on M1, model: sonnet)

**Worktree:** `phase13-m2` off post-M1 main.
**Goal:** new Welcome layout matches spec §4.1 — 2:1 left/right split, right column halved Updates/Recent.

Deliverables:
1. New `src/core/updates/load.ts`: read `~/.nuka/updates.json`; return `UpdateEntry[]`; never throws (empty array on missing/malformed).
2. New `src/core/session/recent.ts`: `listRecentSummaries(home, limit)`; reads `~/.nuka/sessions/*.json`; returns newest-first `RecentItem[]`.
3. New `src/tui/Welcome/UpdatesPanel.tsx`: framed panel; renders up to 6 entries (version + date heading + up to 4 bullets each, truncated with `…`); empty state shows `(no updates)` in `fgFaint`.
4. New `src/tui/Welcome/RecentPanel.tsx`: framed panel; renders up to 6 items (`↩ <text>` + sub-line `<ts>` in `fgFaint`); empty state shows `(no recent sessions)`.
5. Rewrite `src/tui/Welcome/Welcome.tsx`:
   - Outer row container (`flexDirection="row"`, `gap={1}` if Ink supports, else margin).
   - Left framed panel `flexGrow={2}`, contains centered Logo + NUKA + `<model> · <cwd> <branch>` + `Type / for commands`. Vertical centering via `useTerminalSize` rows minus reserved bottom (Tasks/Prompt/Status zones).

   - Narrow terminal (<100 cols): right column hidden; Welcome takes 100% width.
6. Wire updates + recent into `cli.tsx`: load both at startup, pass to `App` as new optional props `updates: UpdateEntry[]`, `recent: RecentItem[]`. App passes them to `Welcome`.
7. Tests:

   - `test/core/session/recent.test.ts`: empty / multiple sessions ordering.
   - `test/tui/Welcome.harness.test.tsx`: 2:1 split renders; right column hidden on narrow terminal; centered NUKA visible.

Acceptance: vitest green; `npm run build` clean; manual smoke confirms new Welcome on first launch.

### M3 — Status panel two-column + icon/text + expanded context  (depends on M1, model: sonnet)

**Worktree:** `phase13-m3` off post-M2 main.
**Goal:** Status panel matches spec §4.2 — two-column dense layout, expanded context, `/status-hub` toggle.

3. `StatusPanel.tsx` rewrite:
   - **Drop** `cost-time` segment, **drop** `fmtElapsed` and the per-second tick. Add `cost` segment as standalone.
   - Add `inputTokens`, `outputTokens` props; expand `context` segment text to `<bar>  <used>k/<max>k · <pct>% · in:<n>k out:<n>k`.

### M4 — Tasks focus mode + TasksSubmenu  (depends on M3, model: sonnet)

**Worktree:** `phase13-m4` off post-M3 main.
**Goal:** Tab from Prompt → Tasks-focused; `j/k` navigates; Enter opens `tasks` submenu; Esc returns.

Deliverables:
1. Extend `UIState` in `App.tsx` with `{ kind: 'tasks-focused'; cursor: number }`.
2. Extend `SubmenuDescriptor` with `{ kind: 'tasks'; focusItem: number }`.
3. App.tsx Tab handler: when in `normal` and Tasks panel non-empty, transition to `tasks-focused` with cursor 0. From any other UIState, Tab is a no-op.
4. Reducer cases: `tasks-focused` ← `j`/`k`/`↑`/`↓` mutate cursor (clamped); Enter → `submenu(tasks, focusItem=cursor)`; Esc / Tab → `normal`.
5. `TasksPanel.tsx`: when `focused` (new prop indicating UIState `tasks-focused`), border in `primary`; cursor row uses `primaryDeep` background.
6. New `src/tui/Submenu/TasksSubmenu.tsx`: full submenu; reads the same data sources as TasksPanel; renders the focused item's full detail (plan: full title + description; subagent: full label + elapsed + recent stdout tail if available; background: full description + outputFile path). Read-only.
7. Update Prompt footer hint to include `Tab 任务` when Tasks panel is non-empty.
8. Tests:
   - `test/tui/Tasks.focus.test.tsx`: stub a populated Tasks panel; Tab → focused state, `j` moves cursor, Enter → submenu, Esc → normal.
   - `test/tui/Submenu/Tasks.harness.test.tsx`: opens `tasks` submenu, asserts focused item details rendered.

Acceptance: vitest green; build clean.

### M5 — Multi-select / list-editing fields (Providers + Plugins)  (depends on M4, model: opus)

**Worktree:** `phase13-m5` off post-M4 main.
**Goal:** `Providers` and `Plugins` config categories support add/edit/delete (providers) and toggle (plugins).

Deliverables:
1. `Field.tsx`: add `list` field type variant; renders vertical checklist with cursor + space-toggle.
2. New `src/tui/Submenu/config/ProvidersForm.tsx` (replaces `ProviderForm.tsx`):
   - Top: providers list with cursor.
   - Footer: `a 添加 · e 编辑 · d 删除 · ⏎ 设为 active · Esc 关闭`.
   - `e` opens an inline editor (id/baseUrl/apiKey/format/selectedModel) reusing existing Field primitive.
   - `a` opens the same editor with empty values (id required).
   - `d` prompts a one-key `y/n` confirm; on `y`, splice the array and save.
   - `⏎` sets `active.providerId` to the cursor's id.
   - `s` saves all pending edits via `saveConfigPatch`.
3. `PluginsForm.tsx` rewrite to multi-select:
   - List `loadedPlugins` (already passed as prop). Each row: `[ ] <name>` or `[x] <name>` based on whether `name ∈ config.plugins.enabled`.
   - Space toggles. `s` saves.
4. `ConfigSubmenu.tsx`: rename `Provider` category to `Providers`; wire `ProvidersForm` and updated `PluginsForm`.
5. Delete `src/tui/Submenu/config/ProviderForm.tsx` and its test.
6. Tests:
   - `test/tui/Submenu/config/ProvidersForm.test.tsx`: add/edit/delete round-trip; set-active.
   - `test/tui/Submenu/config/PluginsForm.test.tsx`: space toggles enabled list; save persists.
   - Update `ConfigSubmenu.harness.test.tsx` for the renamed category.

Acceptance: vitest green; build clean; manual smoke confirms a provider can be added and selected without leaving the TUI.
